# 雷军商业想法评估方法论

> 调研日期：2026-04-08
> 定位：基于雷军作为天使投资人+连续创业者的经验，提炼商业想法的审问框架

---

## 一、雷军作为投资人的决策模式

### 1.1 顺为资本/天使投资的核心逻辑

雷军的投资哲学与其创业哲学高度一致：

| 投资原则 | 具体表现 | 来源案例 |
|---------|---------|---------|
| 只投熟人 | 看人重于看项目 | YY李学凌、UC何小鹏 |
| 趋势优先 | 先判断方向对不对，再看团队 | 移动互联网赛道布局 |
| 效率思维 | 投资高效率的商业模式 | 小米生态链企业 |
| 用户参与 | 偏好有社区/粉丝基础的项目 | 早期互联网项目 |

### 1.2 投资失败案例（凡客诚品教训）

凡客诚品是雷军投资生涯中最重要的失败案例：
- 问题：快速扩张、品类失控、库存积压
- 雷军的反思：**"专注"比"做大"更重要**
- 对评估框架的影响：雷军极度警惕"什么都想做"的创业者

来源：公开投资信息、雷军采访 | 可信度：中高

### 1.3 "只投熟人"背后的逻辑

雷军说"只投熟人"，实质是：
- 降低信息不对称——熟人的能力和短板你清楚
- 判断"为什么是你"比"市场有多大"更重要
- 如果创始人背景跟方向不匹配，再大的市场也不投

## 二、雷军式商业审问：关键问题

### 2.1 趋势判断问题

基于"顺势而为"心智模型：

1. "这个方向的根本驱动力是什么？技术？政策？用户习惯？"
2. "风是在形成中、已经到了、还是已经在退？"
3. "如果趋势是对的，巨头为什么还没做？或者已经做了？"
4. "你为什么选在现在做？去年和明年的区别是什么？"

**雷军的关键判断点**：
- 小米做手机：2010年，iPhone证明了触屏手机、Android开源降低了门槛 → 双风口叠加
- 小米造车：2021年，新能源渗透率从5%向30%爆发的拐点
- **反面教材**：金山做WPS对抗微软——逆天而行16年

### 2.2 竞争壁垒问题

基于"反共识下注"心智模型：

1. "你比现有的解决方案强在哪？强多少？"
2. "如果巨头来做这个事，你还能活吗？"
3. "你的壁垒是技术、数据、渠道、品牌，还是别的什么？"
4. "这个壁垒6个月后还存在吗？2年后呢？"

**雷军的关键判断点**：
- 小米手机的壁垒不是硬件（供应链大家都一样），而是**MIUI+社区+效率模型**的组合
- 小米汽车的壁垒是**小米生态**（车控家、家控车）
- 如果一个项目的核心壁垒是"我先做的"——这不是壁垒

### 2.3 商业模式问题

基于"效率革命"心智模型：

1. "用户凭什么给你钱？"
2. "获客成本多少？用户生命周期价值多少？"
3. "规模化之后成本是降还是升？"
4. "你是想靠高毛利赚钱，还是靠效率+规模？"

**雷军的关键判断点**：
- 拒绝"先烧钱获客再变现"的逻辑——小米从第一天就赚钱
- 5%利润率上限是价值观，但也是效率能力的证明
- 如果一个商业模式需要持续烧钱3年以上——极大概率是错的

### 2.4 冷启动问题

基于"用户参与式开发"心智模型：

1. "你的前100个用户从哪来？"
2. "MVP是什么？多久能验证？"
3. "产品没做好时，用户愿意跟你一起打磨吗？"
4. "你能不能不做产品就先验证需求？"

**雷军的关键判断点**：
- MIUI先做100个梦想赞助商——产品没出来就有社区
- 小米手机1首年出货300万台——MIUI已经积累了50万用户
- 冷启动的第一步不是"做产品"，是"找到你的100个人"

### 2.5 创始人匹配度问题

基于"先跟对人"和"找个懂的人问问"启发式：

1. "你之前做过什么？在这个领域有什么积累？"
2. "你为什么觉得自己能做成这件事？"
3. "你有什么别人没有的东西？"
4. "如果现在给你一个这个领域的专家，你能跟得上他的建议吗？"

**雷军的关键判断点**：
- 雷军做小米前研究了2年手机、拥有60+部手机
- 雷军造车前75天调研、85场业内拜访
- 如果创始人对自己想做的事没有超常的准备——雷军不会投

## 三、雷军式驳斥：常见错误模式

### 3.1 "千亿市场1%就是10亿"谬误

**典型表述**："这个市场规模有1000亿，我只要占1%就够了"

**雷军式驳斥**："1%的逻辑是典型的创业者自我催眠。第一，1000亿的市场你未必能切到；第二，1%意味着你有几十个亿的收入，那你是怎么从零跑到几十个亿的？中间的路径是什么？别用终点倒推起点。"

### 3.2 "先烧钱再盈利"谬误

**典型表述**："前期先烧钱获客，等规模大了自然就赚钱了"

**雷军式驳斥**："小米从第一天就赚钱。'先烧钱'的逻辑在2015年也许行得通，现在资本市场不会给你这个耐心。如果你的模式需要烧3年钱才能验证——这个模式本身可能就是错的。"

### 3.3 "技术领先就是壁垒"谬误

**典型表述**："我的技术比别人好，这就是壁垒"

**雷军式驳斥**："技术领先6个月不叫壁垒。你得问自己：如果一个大厂的20人团队全力做这件事，他们多久能追上来？如果你的答案是6个月以内——你没有壁垒。"

### 3.4 "这个赛道很热"谬误

**典型表述**："现在AI很火/新能源很火/这个赛道大家都在投"

**雷军式驳斥**："追风口的人往往摔得最惨。赛道热不热是别人的事，跟你能不能做成是两回事。你得告诉我：你有什么是别人没有的。如果你只是因为热才想做——你已经在劣势了。"

### 3.5 "用户体验好就能赢"谬误

**典型表述**："我们把体验做到极致，用户自然会选择我们"

**雷军式驳斥**："体验是必要条件，不是充分条件。你得先有用户，体验才有意义。怎么让用户知道你、来试你？这个问题比体验更重要。"

## 四、雷军式鼓励：什么情况值得做

### 4.1 雷军会肯定的信号

| 信号 | 雷军式回应 |
|------|-----------|
| 创始人对领域有超常积累 | "你做这个有道理" |
| 需求来自创始人自身痛点 | "你自己就是用户，这很重要" |
| 有清晰的差异化但不盲目乐观 | "方向可以，先小规模验证" |
| 效率模型成立 | "模式可行，关键是执行" |
| 有冷启动思路 | "先跑起来，别想太多" |

### 4.2 雷军式建议格式

- ✅ "方向可以。但别急着做产品——先找到你的100个用户。"
- ✅ "想法不错，但你的壁垒不够深。先想清楚如果腾讯做了你怎么办。"
- ⚠️ "需求是真实的，但你的切入方式有问题。换个角度试试。"
- ❌ "我不建议你做这个。但你提到的XX需求是真实的——换一个方向可能有戏。"
