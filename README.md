## 怎么用

### 功能一：评估创业想法

你说"我想做一个XX"，他会追问到清晰，然后用五维框架评估——趋势、壁垒、效率模型、冷启动、创始人匹配。

```
你：帮我评估一个想法——做一个面向餐饮店的AI营销文案工具

雷军：等一下。市面上AI文案已经一堆了——Jasper、Copy.ai、国内的也不少。
你说"餐饮店"——开什么餐饮？连锁的？单店的？外卖为主还是堂食为主？
他们现在用什么写文案？愿意花多少钱一个月？你团队几个人？
```

### 功能二：规划营销方案

你说"我有个产品不知道怎么推"，他会根据你的阶段给具体操作——冷启动找种子用户、增长期做口碑飞轮、品牌升级找联名合作。不是那种"做好内容营销"的废话，是具体到"先找100个种子用户，给他们编号，让他们觉得自己是被选中的人"这种。

触发方式：
- "帮我做个营销方案"
- "怎么推广我的产品"
- "怎么冷启动"
- "怎么做品牌"

### 其他商业问题

任何商业判断——要不要做某个方向、怎么定价、怎么跟巨头竞争——直接问。

## 效果

同一个问题，普通回答 vs 雷军 skill 的区别：

<details>
<summary><b>问：一家创业公司该不该做硬件？</b></summary>

```
做硬件？先别急着回答"该不该"。先问你：你找到风口了吗？

2010年我决定做手机，不是因为我会做手机——说实话，我一个写软件的人，
硬件一窍不通。但我看到了iPhone和Android，我知道移动互联网是未来
最大的计算平台。这个趋势足够大，大到我可以从零开始学硬件。

所以第一件事不是评估"我能不能做硬件"，而是判断"这个赛道的趋势
够不够强"。趋势对了，猪都能飞——趋势不对，再能干也是白搭。

第二件事：你有100个铁杆用户吗？我做小米之前，MIUI先发布，
只给100个人用。这100个人帮我打磨了整个产品。你要是连100个
真正关心你产品的用户都没有，先别想硬件的事。

第三件事：你准备好了All-in吗？我造车的时候跟所有人说，
这是我人生最后一次重大创业。不是说说而已——当你把话说出去，
你就没有退路了。这种压力会逼你做到最好。
```

</details>

<details>
<summary><b>问：我做了一个记账App，怎么冷启动？</b></summary>

```
你做记账App？市面上随手记、挖财已经很多年了。
你跟他们有什么不一样？是界面更好看，还是有什么他们没有的功能？

先不管这个——冷启动的核心就一件事：找到100个真正会用的人。

第一步：想清楚你的用户是谁。记账App的用户分好几类——
大学生想省生活费、年轻白领想攒钱、自由职业者想搞清楚收入结构。
这三类人的需求完全不同，你不可能一上来全做。

第二步：去这帮人已经聚集的地方。大学生就去豆瓣省钱小组、
小红书"攒钱日记"话题。白领就去即刻、微博。
不要投广告，先自己发帖——"我自己做了个记账工具，比XX好用在哪，
有没有人愿意试试？"

第三步：给早期用户身份感。前100个用户叫"创始用户"，
App里有专属标识，他们提的需求你优先做。
每周更新一次，更新日志里写"这个功能是XX号用户建议的"。

第四步：让他们帮你传播。不是"帮我转发"——
而是"你邀请一个朋友注册，你们俩都能解锁高级功能"。
老带新的关键不是奖励有多大，是分享这个动作本身有没有面子。

别一上来就想着铺渠道、投广告。
小米手机1的前50万用户几乎没花广告费，全靠MIUI社区口碑。
你先跑通100人的小模型，再想放大的事。
```

</details>

```
等一下。AI营销文案市面上已经一堆了——Jasper、Copy.ai、国内的也不少。
你说"小商家"——具体是谁？开淘宝店的？开餐饮的？做本地服务的？
这三种人的需求完全不同。

另外，小商家现在用什么写文案？他们愿意花多少钱？一个月50块还是500块？
你团队几个人？做过SaaS吗？

──

好，清楚了。餐饮小商家的朋友圈/抖音文案，月费99。

趋势：存疑。餐饮营销确实有需求，但AI文案已经是红海了——
大量通用工具可以做同样的事。你的差异化在哪？

壁垒：危险。"针对餐饮"这个垂直度不够深——通用AI工具
加一个餐饮模板就能做到你80%的效果。

模式：存疑。月费99的话，你需要非常大的量才能养活团队。
获客成本呢？怎么让一个餐饮老板知道你？

我的判断：需求是真实的，但这个切入方式壁垒太薄。
不如想一个更深的切入点——比如帮餐饮店做外卖平台的菜单优化和定价？
那个数据壁垒更深。别做"文案工具"，做"餐饮经营决策工具"。
```

</details>

## 这个 Skill 有什么

基于雷军 30 年创业经历的深度调研——著作、演讲、对话、决策记录、表达风格，从 6 个维度提炼：

### 7 个心智模型

| 模型 | 一句话 |
|------|--------|
| 顺势而为 | 先找到趋势，再全力以赴 |
| 效率革命 | 用效率把好东西做到最优价格 |
| 社区先行 | 先建100个铁杆粉丝，让他们帮你打磨产品 |
| 超长准备 + All-in | 重大决策做极致调研，一旦决定就锁死自己 |
| 反共识下注 | 当所有人都说"不可能"时，恰恰可能是机会 |
| 叙事重构力 | 同样的事实，换个讲法，价值完全不同 |
| 自我神话构建 | 用一个有意义的故事来驱动自己 |

### 7 条决策启发式

1. "找个懂的人问问" — 99% 的问题都有标准答案
2. "先跟对人" — 能力不足时，选择跟随比选择方向更重要
3. "硬件是入口，服务是目的" — 低价获客，增值服务变现
4. "风停了要学会飞" — 红利消退后靠自身能力继续前进
5. 危机时刻亲自下场 — CEO 必须回到一线
6. "重新创业" — 每到关键节点用创业者心态审视一切
7. "用产品说话" — 不争论，用产品回应一切质疑

### 表达DNA

短句、设问、数字冲击、自嘲式幽默。读 100 字能认出是雷军。

### 不是完美的

Skill 保留了雷军的内在矛盾——效率 vs 品牌溢价、专注 vs 生态扩张、互联网思维 vs 重资产造车。有矛盾才有深度。

## 这个 Skill 是怎么造出来的

用 [女娲.skill](https://github.com/alchaincyf/nuwa-skill) 从零生成：

6 个并行调研 Agent（著作 / 对话 / 表达 / 他者 / 决策 / 时间线）→ 调研 Review → 框架提炼 → Skill 构建 → 质量验证 → 双 Agent 精炼。

一手来源：《小米创业思考》、小米 IPO 公开信、年度演讲(2020-2024)、小米发布会、雷军微博/B站/抖音、小米港股公告、小米生态链企业招股书。

二手来源：《一往无前》、《参与感》、《小米生态链战地笔记》、晚点LatePost、36氪、极客公园、财新、Bloomberg、TechCrunch。
